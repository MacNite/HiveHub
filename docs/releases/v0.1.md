# HiveHub V0.1 — first release 🐝

HiveHub's first official release! **All published PCBs are tested and working**, the firmware runs stably in the field, and the self-hosted backend is feature-complete for day-to-day monitoring — time to draw a line and call it **V0.1**.

## What is HiveHub?

HiveHub is an open-source, ESP32-based data collector for beehive sensors and scales. One device gathers **weight, temperature, humidity, sound, vibration, entrance traffic, power state, and network state** from up to **18 hives** and sends the readings to a self-hosted FastAPI + PostgreSQL backend, where they can be viewed in [HivePal](https://github.com/martinhrvn/hive-pal) or the built-in web dashboard.

## Highlights of this release

- **Tested hardware.** Every PCB published in `pcb-design/` has been fabricated, assembled, and verified working.
- **New recommended platform: ESP32-C6 + NAU7802.** The compact Seeed XIAO **ESP32-C6 Scale Module (V0.4)** is now the recommended main controller. It reads 2 scales through its on-board **NAU7802** 24-bit I2C ADC and pairs with up to 18 BLE in-hive sensors. The **NAU7802 breakout PCB (v0.2)** expands a HiveHub to up to **16 wired scales** (8× NAU7802 behind a TCA9548A mux) — or, populated with its own XIAO MCU, works as a standalone BLE scale sensor.
- **Multi-hive firmware.** A dynamic registry of up to 18 hives, each with one scale source (NAU7802, legacy HX711, or a BLE HiveScale) and one in-hive sensor, all configured from the on-device provisioning portal.
- **Wireless in-hive sensors.** HolyIot 25015, RuuviTag, HiveInside ESP32-C6, and beehivemonitoring.com HiveHeart/HiveScale over BLE/GATT — temperature, humidity, pressure, and vibration with no wiring.
- **Robust field operation.** Deep-sleep duty cycle, DS3231 RTC timekeeping, SD-card caching and backfill, OTA firmware updates with per-board images (`esp32` / `esp32-c6`), and off-grid solar/battery telemetry (INA219, MAX17048).
- **Self-hosted backend.** FastAPI + PostgreSQL with API-key device auth, HivePal integration, rule-based colony insights, backend load-cell temperature compensation, optional MQTT/Home Assistant bridge, and an optional built-in dashboard — deployed with Docker Compose.

## Recommended setup

1. Fabricate/assemble the **ESP32-C6 Scale Module (V0.4)** — off-the-shelf modules on pin headers, no SMD soldering for the core build.
2. Need more than 2 wired scales? Add the **NAU7802 breakout PCB** on the I2C expansion header (up to 16 scales).
3. Flash the firmware with `pio run -e xiao_esp32c6` and configure everything from the on-device provisioning portal.
4. Stand up the backend with Docker Compose ([docs/docker-install.md](../docker-install.md)) and claim the device in HivePal or the built-in dashboard.

See [pcb-design/README.md](../../pcb-design/README.md) for board details and the [main README](../../README.md) for the full setup guide.

## Deprecation notes

- The **ESP32 30-pin Scale Module** (HX711/INMP441-based) is obsolete. Existing boards keep working and the `esp32dev` firmware target remains, but no further board revisions are planned — use the ESP32-C6 Scale Module for new builds.
- The separate **Power Module** will probably be discontinued: the C6 Scale Module carries its own buck-boost regulator and battery fuel gauge, so most builds can power it directly.
