Weigh two beehives.
Self-hosted. Open source.

HiveScale is an ESP32-based dual beehive scale system. It tracks weight, temperature, humidity, in-hive sound and vibration, and power state for two hives, and sends everything to your own FastAPI + PostgreSQL backend.

🚀 Get started ⚙️ Config tool ★ Star on GitHub

What it measures

Two independent hives, one ESP32. Every sensor below is optional and compiled in per device — start with weight and add the rest as you build.

⚖️

Dual load cells

Two HX711 amplifiers drive two independent hive scales for per-colony weight trends.

🌡️

Per-hive temperature

DS18B20 probes on a shared 1-Wire bus, or an in-hive BLE sensor as an alternative source.

🔊

In-hive sound

Stereo INMP441 I2S microphones with per-band FFT energy to catch piping and stress.

📳

Comb vibration

An in-hive BLE sensor (e.g. HiveInside) captures the ~20 Hz pre-swarm signal microphones miss.

💧

Ambient T & humidity

An SHT4x on I2C records weather context alongside the hive readings.

🛰️

Off-grid power telemetry

Optional INA219 solar and MAX17048 LiPo fuel-gauge data for unattended deployments.

📶

BLE in-hive sensor

Pair a HolyIot 25015 beacon (temp / humidity / pressure / accel) — no extra wiring.

💾

SD cache & backup

Local buffering when uploads fail, plus an append-only persistent measurement backup.

🐝

BeeCounter ready

Optional entrance traffic counting of incoming and outgoing bees over I2C.

🔑

Claim-code pairing

Claim devices from HivePal without touching the database — just a pairing code.

📡

OTA & remote commands

Over-the-air updates, remote calibration, reboot, Wi-Fi reset and config versioning.

🧠

Rule-based insights

The backend auto-evaluates weight, temperature, sound and vibration into colony insights.

Hardware at a glance

A mains-or-solar ESP32 dev board plus off-the-shelf modules on pin headers — no SMD soldering required for the core build.

Core

ESP32 dev board · 2× HX711 + load cells · 2× DS18B20 probes · 2× INMP441 mics · SHT4x · DS3231 RTC · microSD module · setup button · 3.3 V supply.

Optional off-grid

INA219 solar telemetry · MAX17048 LiPo gauge · CN3791 solar charger · TPS63020 buck-boost · TP4056 · LiPo + solar panel.

Boards

KiCad Scale Module (central, header-based) and a Power Module (solar / battery / LTE) connected over I2C / ESP-NOW. Fabrication outputs included.

Full pin mapping, wiring diagrams and the BOM live in the repository README and docs/wiring.md.

Set up your own HiveScale

Five steps from clone to live data. The firmware is a PlatformIO project; the backend is a Docker Compose stack.

1

Get the code

Clone the repository and open the firmware project in PlatformIO (VS Code extension or CLI).

git clone https://github.com/MacNite/HiveScale.git
cd HiveScale
2

Stand up the backend

Bring up the FastAPI + PostgreSQL stack with Docker Compose. The API listens on port 31115.

cd docker
cp .env.example .env
# edit API_KEY, HIVEPAL_SERVICE_API_KEY, DB password, volumes
docker compose up -d

See docker-install.md or truenas-install.md for hosting details.

3

Configure the device

Recommended: configure the device from its on-device provisioning portal. After flashing, hold the setup button to start the HiveScale Wi-Fi access point, connect to it, and set Wi-Fi, backend, hives and sensors right on the device. Those settings live in the device's storage and survive OTA updates.

Optionally pre-seed a brand-new device before first boot with the in-browser config tool (it generates an optional secrets.h) — or skip it and pair everything from the portal.

⚙️ Open the config tool

Manual route: cp firmware/include/secrets.example.h firmware/include/secrets.h and edit it.

4

Flash the firmware

Drop your secrets.h into firmware/include/, then build and upload.

cd firmware
pio run --target upload
pio device monitor   # 115200 baud
5

Claim & watch the data

On first boot the device sends measurements with its claim code. Claim it from HivePal with POST /api/v1/app/devices/claim, then explore live charts and rule-based insights.

Field configuration without reflashing: press the setup button (GPIO27) to open the HiveScale-Setup-XXXX Wi-Fi provisioning portal at http://192.168.4.1.
